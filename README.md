# ShiMetaPi-Pico-G1 SDK

> **Version:** V1.0.0
> **Chip:** GK7206V11A / GK7206V12A
> **Board:** ShiMetaPi-Pico-G1 Series

The ShiMetaPi-Pico-G1 SDK is a secondary development based on the official SDK provided by Guoke Micro (GK). It is adapted and optimized for the hardware characteristics and application scenarios of the ShiMetaPi-Pico-G1 series development boards, aiming to provide developers with a more efficient, convenient, and user-friendly development experience.

---

## 1 Quick Start

### 1.1 Install Host Dependencies

The SDK includes a cross-compilation toolchain and firmware packaging tools, but the following host dependencies are still required:

```bash
sudo apt update
```

```bash
sudo apt install -y build-essential make gcc fakeroot bc bison flex libncurses5-dev libssl-dev libelf-dev zlib1g-dev liblzma-dev liblzo2-dev liblz4-tool libzstd-dev device-tree-compiler
```

### 1.2 Get the SDK

* GitHub

```
git clone https://github.com/ShiMetaPi/shimetapi_pico_g1.git
```

* Gitee

```
git clone https://gitee.com/ShiMetaPi/shimetapi_pico_g1.git
```

```
│   ├── xm7206v11a/        # ShiMetaPi-Pico-G1-128M_DDR

│   └── xm7206v12a/
│       ├── image/         # Firmware images (JFFS2/YAFFS2/EXT4/UBIFS)
│       ├── rootfs/        # Root filesystem
│       ├── lib/           # SDK libraries (static/ + share/)
│       ├── ko/            # Kernel modules
│       └── linux-5.10.y/  # Kernel build artifacts
├── sample/                # Official sample programs (27 categories)
├── source/                # Source code
│   ├── bootloader/        # U-Boot (2020.01)
│   ├── kernel/            # Linux Kernel (5.10.y / 4.9.y)
│   ├── gmp/               # Media processing library (pre-built)
│   ├── rootfs/            # Root filesystem build
│   ├── component/         # System components
│   └── initramdisk/       # Initramdisk
├── tools/                 # Tools and toolchains
│   └── linux/toolchains/  # Cross-compilation toolchain (built-in)

└── tar_rootfs.sh          # Rootfs packaging script
```

### 1.4 run.sh Usage Guide

#### 1.4.1 Select Board Configuration

> This development board is available in two chip versions — GK7206V12A and GK7206V11A — to meet the needs of different application scenarios. The GK7206V12A version is equipped with 256MB DDR, while the GK7206V11A version comes with 128MB DDR.

```bash
./run.sh lunch
```

Interactively select the board and kernel version: (Enter the number as prompted and press Enter to confirm)

```
========================================
  Select Board Configuration
========================================

Board:
  1. ShiMetaPi-Pico-G1-128M_DDR(GK7206V11A)
  2. ShiMetaPi-Pico-G1-256M_DDR(GK7206V12A)

Select board [1-2]: 2
  -> ShiMetaPi-Pico-G1-256M_DDR(GK7206V12A)

Kernel:
  1. linux-5.10
  2. linux-4.9

Select kernel [1-2]: 1
  -> linux-5.10

========================================
  Board:    ShiMetaPi-Pico-G1-256M_DDR(GK7206V12A)
  Kernel:   linux-5.10
========================================
Configuration written to cfg.mk

Done.
```

The selection result will be written to `cfg.mk`.

#### 1.4.2 Build Commands

| Command | Description |
|---------|-------------|
| `./run.sh uboot`  | Build U-Boot bootloader |
| `./run.sh kernel` | Build Linux kernel |
| `./run.sh rootfs` | Build root filesystem |
| `./run.sh driver` | Build kernel drivers (GMP) |
| `./run.sh sysdrv` | Build uboot + kernel + rootfs |
| `./run.sh osal`   | Build OSAL adaptation layer |
| `./run.sh media`  | Build media libraries |
| `./run.sh sample` | Build MPP sample applications |
| `./run.sh mcu`    | Build MCU firmware |
| `./run.sh all`    | Build all components |

| `./run.sh clean kernel` | Clean kernel |
| `./run.sh clean rootfs` | Clean rootfs |
| `./run.sh clean driver` | Clean drivers |
| `./run.sh clean sysdrv` | Clean uboot/kernel/rootfs |
| `./run.sh clean media` | Clean media libraries |
| `./run.sh clean sample` | Clean MPP sample applications |
| `./run.sh clean osal` | Clean OSAL adaptation layer |
| `./run.sh clean mcu` | Clean MCU firmware |
| `./run.sh clean all` | Clean all components |

#### 1.4.4 Information Commands

| Command | Description |
|---------|-------------|
| `./run.sh info` | Display current configuration |
| `./run.sh check` | Check build environment |
| `./run.sh env` | Display toolchain paths |
| `./run.sh list` | List available board configurations |
| `./run.sh help` | Display help information |

#### 1.4.5 Parallel Build

All build commands support the `-j[N]` parameter:

```bash
./run.sh kernel -j       # Use all available cores
./run.sh kernel -j4      # Use 4 cores
./run.sh kernel -j1      # Use 1 core — for debugging
```

## 2 Sample Applications (app_sample/)

| Example | Description |
|---------|-------------|
| `app_sample/face_recognize` | Face detection and recognition (MTCNN + NPU, ~8fps) |
| `app_sample/rpi_detector` | Motion detector |

### 2.1 Build Example Application

```bash
cd app_sample/face_recognize
make
```

### 2.2 Run on the Board

* On the board, run:
```bash
mkdir sd_card
mount /dev/mmcblk1p1 /sd_card
```

* From the SDK host, run:
```
scp app_sample/face_recognize/face_recognize root@<board_ip>:/sd_card/
```

* On the board, run:
```bash
cd /sd_card
chmod +x face_recognize
./face_recognize
```

## 3 FAQ

### 3.1 Build error: `arm-gcc12.2.0-linux-uclibceabi-gcc: command not found`

Make sure the PATH is set via `./run.sh` or `source build/env.sh`. The toolchain is built into `tools/linux/toolchains/`.

### 3.2 How to switch chip model

```bash
./run.sh lunch    # Re-select the board
```

### 3.3 How to check current configuration

```bash
./run.sh info
```

### 3.4 My application is not in rootfs after build

The SDK does not automatically include applications from `app_sample/` into the rootfs. You need to copy them manually:

```bash
cp app_sample/face_recognize/face_recognize out/xm7206v12a/rootfs/usr/bin/
```

Then regenerate the rootfs image:

```bash
./run.sh rootfs
```

### 3.5 How to repackage rootfs

```bash
./run.sh rootfs
```

This command rebuilds the rootfs directory and packages it into the corresponding firmware image format.
